Fourteenth beta release of Loom — an evidence operating system for civil-rights legal teams that turns mixed field evidence (video, photos, audio, FOIA returns) into a defensible event timeline where every claim traces back to source material. Civil-liberties tooling: no face recognition, no suspicion scoring, no automated identity resolution.

> **Beta scope:** prepared for first use in Illinois (state and federal § 1983 actions). Defaults are documented in [`docs/requirements.md`](docs/requirements.md) and are jurisdiction-tunable.
>
> **Not legal advice.** Statutes and rules of evidence are referenced for design context only. Admissibility, consent, retention, and disclosure decisions for any specific matter belong with licensed counsel.

## Release focus

v0.1.14 made ingest work on the desktop, but the *viewing* side was broken: PDFs wouldn't preview or download, and an ingested video wouldn't play. The cause was twofold — the asset download-url returned a `loom://` loopback scheme the webview cannot load in `<video>`/`<img>`/`<object>`/`<a download>`, and the frontend was calling `/download` while the backend served `/download-url`. The latter is the third frontend↔backend route mismatch to ship in as many releases (after `/assets` vs `/assets/upload` and the `useAsset` 404), so this release also adds a guard that makes the whole class fail CI. On-device AI enrichment (transcription, OCR, scene detection producing real results) is the next release, v0.1.16.

## Highlights

- **Assets are served over HTTP on Desktop Lite, so preview/download/playback work (`backend/src/loom/api/v1/storage.py`, `services/storage_backends/local.py`, `frontend/src/components/asset/asset-viewer.tsx`).** The sidecar now exposes a signed byte-serving endpoint (`/api/v1/storage/object/...`); the download-url returns an `http://127.0.0.1:8000/...` URL whose query-string HMAC signature is the credential (media elements can't send auth headers). It supports HTTP Range, so video seeks; sets the right content-type, so PDFs render inline via `<object>` with a download fallback; and cross-origin downloads are forced with an attachment disposition. The Tauri CSP is widened to allow the sidecar origin for media/img/object. The server profile is unchanged — it still hands out Minio https URLs.
- **A route-contract guard now fails CI on drift, and three latent mismatches it caught are fixed (`backend/tests/integration/test_route_contract.py`).** It extracts every literal API path the frontend calls and asserts each resolves against the FastAPI route table — something the mocked hook tests never did. It surfaced that scene detection posted to `/detect-scenes` (route: `/scenes/detect`) and conflict resolution used `/events/{id}/resolutions` (routes: `/events/{id}/conflicts/resolve` and `/conflicts/resolutions/{id}`); all are corrected.
- **A real-SQL Lite end-to-end test now covers asset serving (`backend/tests/integration/test_lite_asset_serving_sql.py`).** Upload a PDF → fetch the download-url → GET it → assert 200, `application/pdf`, byte-for-byte content, a Range request returning 206, and a tampered signature returning 403. This is the gap that let the viewing breakage reach users.
- **The Lite-only presigned-upload completion endpoint is guarded.** It scanned Minio (`list_objects`), which is `None` on Lite; it now returns 404 there instead of risking a 500. Desktop uploads use `POST /upload` directly.
- **New capability matrix doc (`docs/lite-capability-matrix.md`)** records what works on Lite vs server and why, referenced from the desktop guide.

## Migrations

- No Alembic revisions, schema, or configuration changes in v0.1.15. Existing installs upgrade in place with no data migration and no factory reset. Assets uploaded under v0.1.14 are now viewable/downloadable with no reprocessing.

## Install

- **Desktop Lite (individuals):** download the installer for your platform from this release's assets, or follow [`docs/desktop-lite.md`](docs/desktop-lite.md) to build from source. No factory reset required.
- **Server (organizations):** see [`docs/prod-deploy.md`](docs/prod-deploy.md). Rebuild or re-pull both images; server-profile asset serving (Minio presigned URLs) is unchanged.

## Known limitations in 0.1.15

- **On-device AI enrichment is still pending (v0.1.16).** Transcription, OCR, and scene detection run but produce empty/degraded results on the desktop build because the local models and the `ffmpeg` binary are not yet bundled; video proxies/thumbnails are likewise skipped. The *original* media now previews, downloads, and plays — this limitation is only about derived AI output. v0.1.16 adds on-device models (default) plus an opt-in bring-your-own-key cloud path.
- **Inline PDF preview depends on the platform webview.** Windows (WebView2) and macOS (WebKit) render PDFs inline; where the webview lacks a PDF viewer, the panel falls back to a download link.
- **Three breaking-major frontend upgrades held for dedicated migration:** Vite 8, Tailwind CSS 4, ESLint 10. Carryover from v0.1.13.
- **pyo3 advisory suppressed, not resolved** (temporalio's vendored bridge). Carryover from v0.1.6.
- **macOS Intel installer is still not produced** (Apple Silicon only). Carryover from v0.1.1.
- **Desktop sidecar cold-start on Windows can still take 15–25 s.** Carryover from v0.1.3.
- **No E2E browser tests in CI yet** (#112) — carryover; this release's fixes are guarded by unit and integration tests.

## Security

- No new runtime dependencies. Desktop asset URLs are signed with the per-install storage secret and served only from the loopback sidecar (no external exposure); the signature, not a session, authorizes a read, and links expire. The Tauri CSP was widened only to the sidecar's own `http://127.0.0.1:8000` origin for media/img/object/frame. The documented suppressions remain CVE-2025-3000 (torch, local-only) and the temporalio-vendored pyo3 advisory in `.trivyignore`.
- See [`docs/security.md`](docs/security.md) for the threat model and posture.
- Report vulnerabilities privately by emailing the maintainer — do not open a public issue.
