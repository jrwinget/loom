Security hotfix for v0.1.6 — an evidence operating system for civil-rights legal teams that turns mixed field evidence (video, photos, audio, FOIA returns) into a defensible event timeline where every claim traces back to source material. Civil-liberties tooling: no face recognition, no suspicion scoring, no automated identity resolution.

> **Beta scope:** prepared for first use in Illinois (state and federal § 1983 actions). Defaults are documented in [`docs/requirements.md`](docs/requirements.md) and are jurisdiction-tunable.
>
> **Not legal advice.** Statutes and rules of evidence are referenced for design context only. Admissibility, consent, retention, and disclosure decisions for any specific matter belong with licensed counsel.

## Release focus

v0.1.7 is a single-purpose security hotfix on top of v0.1.6. An OpenSSL advisory published 2026-06-09 ([openssl-library.org/news/secadv/20260609](https://openssl-library.org/news/secadv/20260609.txt)) affects the statically-linked OpenSSL inside `cryptography`'s PyPI wheels prior to 48.0.1 (GHSA-537c-gmf6-5ccf). `cryptography` ships transitively in the backend via `pyjwt[crypto]`; v0.1.6 resolved it at 46.0.7, which is vulnerable. This release pins the resolution forward to a fixed line. There are no other changes — no feature surfaces, no schema migrations, no API changes.

## Highlights

- **cryptography 46.0.7 → 49.0.0.** Clears GHSA-537c-gmf6-5ccf by moving past the 48.0.1 fix to the current release line. `cryptography` is an indirect dependency (we declare no direct constraint), so the bump is confined to `backend/uv.lock`; the JWT, MFA/TOTP, and recovery-code paths that exercise it are covered by the existing backend test suite and continue to pass unchanged.

## Migrations

- No new Alembic revisions in v0.1.7. No schema, data, or configuration changes. Existing v0.1.6 desktop installs upgrade in place; the only on-disk artefact this release touches is the sidecar binary, which now bundles the patched `cryptography` wheel.

## Install

- **Desktop Lite (individuals):** download the installer for your platform from this release's assets, or follow [`docs/desktop-lite.md`](docs/desktop-lite.md) to build from source. Existing v0.1.6 installs upgrade in place.
- **Server (organizations):** see [`docs/prod-deploy.md`](docs/prod-deploy.md). Rebuild or re-pull the backend image to pick up the patched `cryptography`; no configuration changes are required.

## Security

- Resolves GHSA-537c-gmf6-5ccf (OpenSSL static-link advisory in `cryptography` < 48.0.1). After the bump, `pip-audit` reports no known un-ignored vulnerabilities; the only remaining documented suppressions are CVE-2025-3000 (torch, local-only, no fixed release) and the temporalio-vendored pyo3 advisory carried in `.trivyignore` since v0.1.6.
- All other security posture from v0.1.6 — the CORS allowlist, shutdown-token, parent-pid watchdog, Windows job-object cleanup, and async-safe graceful shutdown — is unchanged.
- See [`docs/security.md`](docs/security.md) for the threat model and posture.
- Report vulnerabilities privately by emailing the maintainer — do not open a public issue.
