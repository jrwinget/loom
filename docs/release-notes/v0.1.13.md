Twelfth beta release of Loom — an evidence operating system for civil-rights legal teams that turns mixed field evidence (video, photos, audio, FOIA returns) into a defensible event timeline where every claim traces back to source material. Civil-liberties tooling: no face recognition, no suspicion scoring, no automated identity resolution.

> **Beta scope:** prepared for first use in Illinois (state and federal § 1983 actions). Defaults are documented in [`docs/requirements.md`](docs/requirements.md) and are jurisdiction-tunable.
>
> **Not legal advice.** Statutes and rules of evidence are referenced for design context only. Admissibility, consent, retention, and disclosure decisions for any specific matter belong with licensed counsel.

## Release focus

With the v0.1.12 styling fixes in place, a user opening a case found there was no way to reach **Assets** to add evidence — the page existed and was routed, but nothing linked to it. An audit of every navigation target against the route table found this was one of a cluster of reachability gaps: several built case pages (Assets, Timeline, Export) had no UI entry point, the Security/MFA settings page 404'd despite being linked from the account menu and the docs, and Desktop Lite surfaced features it doesn't support. This release gives the case workspace a proper navigation surface, wires the missing route, and gates the Lite-only dead-ends. Frontend-only; no schema or API changes.

## Highlights

- **Every case page is reachable from a case toolbar (`frontend/src/components/layout/case-layout.tsx`, `frontend/src/app.tsx`).** Opening a case now shows a toolbar across the top — **Overview**, **Assets**, **Timeline**, **Conflicts**, **Clusters**, **Map**, **Export** — with a **‹ Cases** link back to the list. Previously Assets (the evidence-upload page), Timeline, and Export had no link anywhere and could only be reached by typing the URL; Conflicts/Clusters/Map lived in the global sidebar with no case context. The case sub-routes are now nested under a shared layout, so the toolbar and the case name/status travel with you across every section.
- **Security settings are reachable again (`frontend/src/app.tsx`).** The account menu's **Settings** link and the docs both point to "Settings → Security" for enabling MFA, but `/settings/security` was never registered as a route, so it fell through to the 404 view. The page itself was complete — it is now wired up.
- **Desktop Lite no longer offers what it can't do (`frontend/src/components/layout/sidebar.tsx`, `frontend/src/routes/cases/[caseId]/index.tsx`).** Lite is single-user and has no organizations or plugin marketplace, but the sidebar still showed an **Organizations** link and pointed **Settings** at the plugins page, and every case showed an empty **Members** tab. On Lite, Organizations and the Members tab are now hidden and **Settings** opens **Security**; the server profile is unchanged.
- **Fixed a dead "Model card" link (`frontend/src/components/review/why-popover.tsx`).** The AI-provenance popover linked to an in-app `/docs/…` path that isn't a route; it now opens the canonical model-card documentation.
- **Regression coverage added.** New tests assert the case toolbar links to every section and marks the active one, that `/settings/security` resolves to the Security page (not the 404 view), and that the Lite profile hides Organizations, the Members tab, and points Settings at Security.

## Migrations

- No Alembic revisions, schema, or configuration changes in v0.1.13. This is a frontend navigation and documentation release; existing installs upgrade in place with no data migration and no factory reset.

## Install

- **Desktop Lite (individuals):** download the installer for your platform from this release's assets, or follow [`docs/desktop-lite.md`](docs/desktop-lite.md) to build from source. No factory reset required.
- **Server (organizations):** see [`docs/prod-deploy.md`](docs/prod-deploy.md). Rebuild or re-pull the frontend image; the backend image is unchanged and no configuration changes are required.

## Known limitations in 0.1.13

- **Three breaking-major frontend upgrades held for dedicated migration:** Vite 8, Tailwind CSS 4, and ESLint 10 each require non-trivial migration work and are tracked as open PRs rather than rushed into this release. The esbuild advisory is gated behind the Vite 8 upgrade.
- **pyo3 advisory suppressed, not resolved.** GHSA-36hh-v3qg-5jq4 lives in temporalio's vendored Rust bridge and cannot be fixed without an upstream temporalio release; the Trivy ignore is removed as soon as one ships against pyo3 ≥ 0.29. Carryover from v0.1.6.
- **No "rotate my codes" surface yet.** Carryover from v0.1.2.
- **macOS Intel installer is still not produced.** The CI matrix builds only `macos-latest` (Apple Silicon). Carryover from v0.1.1.
- **Desktop sidecar cold-start on Windows can still take 15–25 s** under the PyInstaller `--onefile` extract-to-temp path with active antivirus. Carryover from v0.1.3.
- **No E2E browser tests in CI yet** (#112) and **frontend/workflow coverage gaps** (#113) — carryover. This release's navigation fixes are guarded by unit tests rather than a full browser E2E.

## Security

- No new dependencies and no posture change — the fixes are frontend navigation and documentation only. Enabling MFA from the account menu now works (the Security settings page was previously unreachable). The documented suppressions remain CVE-2025-3000 (torch, local-only, no fixed release) and the temporalio-vendored pyo3 advisory carried in `.trivyignore` since v0.1.6.
- See [`docs/security.md`](docs/security.md) for the threat model and posture.
- Report vulnerabilities privately by emailing the maintainer — do not open a public issue.
