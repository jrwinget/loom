Second beta release of Loom — an evidence operating system for civil-rights legal teams that turns mixed field evidence (video, photos, audio, FOIA returns) into a defensible event timeline where every claim traces back to source material. Civil-liberties tooling: no face recognition, no suspicion scoring, no automated identity resolution.

> **Beta scope:** prepared for first use in Illinois (state and federal § 1983 actions). Defaults are documented in [`docs/requirements.md`](docs/requirements.md) and are jurisdiction-tunable.
>
> **Not legal advice.** Statutes and rules of evidence are referenced for design context only. Admissibility, consent, retention, and disclosure decisions for any specific matter belong with licensed counsel.

## Re-cut notice

The original v0.1.1 installers (and the v0.1.0 installers before them) did not boot: the PyInstaller-built desktop sidecar instantiated the FastAPI application and exited without binding a listening socket, so the Tauri shell blocked the OS message pump for 60 seconds waiting on a health endpoint that never answered. On Linux this produced GNOME's "Wait or Force Quit" dialog; on Windows it produced the `(Not Responding)` pinwheel.

This re-cut of v0.1.1 ships the same feature scope as the original beta plus a desktop hotfix:

- The sidecar binary now starts a uvicorn server bound to `127.0.0.1:8000`.
- The Tauri shell no longer blocks the main thread during boot. A "Loom is starting…" panel renders immediately; the app swaps in once the backend answers, or surfaces the sidecar's stderr in a retryable error state if it does not.
- A new CI smoke step runs the freshly-built sidecar on every matrix platform and fails the build if `/api/v1/health` does not answer within 30 seconds. The original regression cannot land silently again.

The original v0.1.1 binaries plus stray v0.1.0 and v0.1.2 binaries that had accumulated on this release page have been removed. The download list below is the only set of installers that should be used.

## Highlights (unchanged from the original v0.1.1 beta)

- **Correlation confidence panel lands in the UI.** The API-only correlation surface from 0.1.0 is now wired into the review workspace: `<CorrelationPanel/>` with accept/reject controls plus a timeline confidence threshold and probable-match badge.
- **Correlation decisions are audit-tracked.** Accepting or rejecting a correlation writes per-asset entries into chain of custody, surfaced in the court bundle.
- **Presigned media everywhere.** Scene thumbnail endpoint returns short-lived presigned URLs instead of raw object paths, matching the rest of the asset surface.
- **Workflow E2E coverage.** Temporal `WorkflowEnvironment` harness with a full ingest e2e that exercises the real workflow lifecycle, not just structural assertions.
- **MFA test coverage.** Hook plus challenge and setup component tests close a gap from 0.1.0.
- **Operational telemetry.** Worker Prometheus metrics with alert rules for the server profile.
- **Accessibility.** Five frontend a11y gaps closed across the review and timeline surfaces.
- **Cross-platform file validation.** Replaced `python-magic` with `puremagic` so desktop builds on Windows and macOS no longer require libmagic.
- **Security.** CVE bumps for mako, pip, python-multipart, urllib3.

## Install

- **Desktop Lite (individuals):** download the installer for your platform from this release's assets, or follow [`docs/desktop-lite.md`](docs/desktop-lite.md) to build from source. Single-user, local-only — nothing leaves the machine unless you export.
- **Server (organizations):** see [`docs/prod-deploy.md`](docs/prod-deploy.md) for the production checklist (TLS, secrets, backup rotation, observability stack).

## Known limitations in 0.1.1

- **macOS Intel installer is not produced.** The CI matrix builds only `macos-latest`, which resolves to Apple Silicon. Tracked for v0.1.2.
- **Desktop sidecar cold-start on Windows can take ~10 s** under the PyInstaller `--onefile` extract-to-temp path with active antivirus. The boot panel makes this visible (no "Not Responding" anymore), but the wait remains. `--onefile` → `--onedir` migration is tracked for v0.1.2.
- **No E2E browser tests in CI yet** (#112) — Playwright smoke flow is tracked post-beta; golden-path browser flows continue to be exercised manually.
- **Frontend test coverage gaps** (#113) — roughly 14 hooks and 11 components without dedicated tests, tracked post-beta.
- **Workflow E2E coverage is partial** — ingest is now exercised end-to-end (#119); correlation, scene, OCR, transcription, export, and url-ingest workflows still rely on structural assertions (#113).

## Security

- See [`docs/security.md`](docs/security.md) for the threat model and posture.
- Report vulnerabilities privately by emailing the maintainer — do not open a public issue.
- One outstanding upstream advisory: CVE-2026-3219 (`pip` ≤ 26.0.1, no upstream patch yet) remains documented as tolerable_risk; pip is build-time only and supply-chain attacks on the upstream toolchain are out-of-scope per the threat model.
