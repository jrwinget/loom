## Release focus

v0.1.6 is a maintenance release that clears the accumulated weekly Dependabot backlog â€” 30 dependency updates across five ecosystems (GitHub Actions, Cargo, npm/pnpm, uv/PyPI) â€” landed together with the small code migrations the breaking majors required. There are no new feature surfaces, no schema migrations, and no intended change to desktop or server runtime behaviour; the surface area is library versions and the code adjustments needed to keep building and passing tests against them. Four CI/tooling bugs surfaced during the sweep were fixed in the same release.

## Also fixed â€” surfaced during the sweep

- **pre-commit frontend hooks.** The local `prettier`/`eslint` hooks ran via `pnpm --dir frontend exec` (cwd `frontend/`) while pre-commit passes repo-root-relative paths, so the tools looked for `frontend/frontend/src/...` and aborted every commit that staged frontend sources. The hooks now strip the leading `frontend/` so paths resolve under the working dir.
- **Windows sidecar smoke cleanup race.** `backend/scripts/smoke_sidecar.py` could fail with `WinError 32` when `TemporaryDirectory` cleanup raced the sidecar's release of the sqlite `loom.db` handle on Windows â€” after all three HTTP contract checks had already passed. Cleanup now passes `ignore_cleanup_errors=True`.
- **Dead `scenedetect[opencv]` extra.** scenedetect 0.7 folded opencv into a core dependency and dropped the `opencv` extra, so `scenedetect[opencv]` resolved with a warning and a silently-ignored extra. The dependency is now plain `scenedetect>=0.7`; opencv-python remains resolved as a core dependency.
- **Trivy pyo3 advisory.** GHSA-36hh-v3qg-5jq4 (pyo3 0.25.1, out-of-bounds read) is vendored inside the temporalio SDK's prebuilt Rust bridge; temporalio 1.28.0 (latest) still pins it and there is no upgrade path on our side. A documented, scoped `.trivyignore` records the advisory with its removal condition rather than weakening the scan.
