Eleventh beta release of Loom — an evidence operating system for civil-rights legal teams that turns mixed field evidence (video, photos, audio, FOIA returns) into a defensible event timeline where every claim traces back to source material. Civil-liberties tooling: no face recognition, no suspicion scoring, no automated identity resolution.

> **Beta scope:** prepared for first use in Illinois (state and federal § 1983 actions). Defaults are documented in [`docs/requirements.md`](docs/requirements.md) and are jurisdiction-tunable.
>
> **Not legal advice.** Statutes and rules of evidence are referenced for design context only. Admissibility, consent, retention, and disclosure decisions for any specific matter belong with licensed counsel.

## Release focus

With onboarding and navigation finally working end to end, the first thing operators saw was a readability bug: the **Reset data** and **Keyboard shortcuts** dialogs rendered with see-through panels, leaving their text floating over a dark scrim. The cause was a missing design token — the `card` surface color those panels reference was never defined in the theme, so the utility emitted no background at all. This release defines the token (and its `popover` sibling) once, which repairs every card and popover surface across the app, not just the two dialogs. It also sweeps a related class of bug — class names concatenated without a separating space — that had been silently voiding badge colors, the active-tab underline, and other styling. Frontend-only; no schema or API changes.

## Highlights

- **Dialogs and surfaces render solid again (`frontend/tailwind.config.ts`, `frontend/src/styles/globals.css`).** `bg-card` is used by roughly two dozen components, but `card` was never registered in the Tailwind theme or its CSS variables, so the class produced no rule and the surface stayed transparent. On flush pages that looked fine against the page background; on the reset-data and keyboard-shortcuts dialogs — which sit over a dark scrim — it made the text unreadable. Defining `card` and `popover` (light and dark) fixes both dialogs and every other card/popover surface in one place.
- **Class names glued together by missing spaces are restored (`frontend/src/components/layout/header.tsx`, `frontend/src/components/asset/asset-detail.tsx`, `frontend/src/routes/cases/[caseId]/index.tsx`, `frontend/src/routes/cases/[caseId]/timeline.tsx`).** Several multi-line `className` strings joined without a trailing space, producing invalid tokens like `rounded-lgborder` and `font-mediumbg-yellow-100` that the browser ignored. That had quietly voided status and processing badge colors (rendering them transparent), the active-tab underline, the timeline detail-panel border, and some button text and hover colors. Every join now carries its space.
- **An onboarding quick-start (`docs/desktop-lite.md`).** A new **Quick start** section walks a first-time operator from sign-in through creating a case, adding evidence, reviewing and annotating, building a timeline, and exporting a court bundle — plus where to find keyboard shortcuts, how to add a second factor, and how to report issues.
- **Regression coverage added.** A theme-token guard asserts `card` stays defined in both the Tailwind config and the backing CSS variables; a header test asserts the shortcuts dialog applies `rounded-lg` and `border` as discrete classes, so the missing-space class of bug cannot silently return.

## Migrations

- No Alembic revisions, no schema changes, and no configuration changes in v0.1.12. This is a frontend styling and documentation release; existing installs upgrade in place with no data migration and no factory reset.

## Install

- **Desktop Lite (individuals):** download the installer for your platform from this release's assets, or follow [`docs/desktop-lite.md`](docs/desktop-lite.md) to build from source. No factory reset required.
- **Server (organizations):** see [`docs/prod-deploy.md`](docs/prod-deploy.md). Rebuild or re-pull the frontend image; the backend image is unchanged and no configuration changes are required.

## Known limitations in 0.1.12

- **Three breaking-major frontend upgrades held for dedicated migration:** Vite 8, Tailwind CSS 4, and ESLint 10 each require non-trivial migration work and are tracked as open PRs rather than rushed into this release. The esbuild advisory is gated behind the Vite 8 upgrade.
- **pyo3 advisory suppressed, not resolved.** GHSA-36hh-v3qg-5jq4 lives in temporalio's vendored Rust bridge and cannot be fixed without an upstream temporalio release; the Trivy ignore is removed as soon as one ships against pyo3 ≥ 0.29. Carryover from v0.1.6.
- **No "rotate my codes" surface yet.** Carryover from v0.1.2.
- **macOS Intel installer is still not produced.** The CI matrix builds only `macos-latest` (Apple Silicon). Carryover from v0.1.1.
- **Desktop sidecar cold-start on Windows can still take 15–25 s** under the PyInstaller `--onefile` extract-to-temp path with active antivirus. Carryover from v0.1.3.
- **No E2E browser tests in CI yet** (#112) and **frontend/workflow coverage gaps** (#113) — carryover. This release's visual fixes are guarded by unit tests rather than a full browser E2E.

## Security

- No new dependencies and no posture change in this release — the fixes are frontend styling and documentation only. The documented suppressions remain CVE-2025-3000 (torch, local-only, no fixed release) and the temporalio-vendored pyo3 advisory carried in `.trivyignore` since v0.1.6.
- See [`docs/security.md`](docs/security.md) for the threat model and posture.
- Report vulnerabilities privately by emailing the maintainer — do not open a public issue.
