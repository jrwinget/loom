Fifteenth beta release of Loom — an evidence operating system for civil-rights legal teams that turns mixed field evidence (video, photos, audio, FOIA returns) into a defensible event timeline where every claim traces back to source material. Civil-liberties tooling: no face recognition, no suspicion scoring, no automated identity resolution.

> **Beta scope:** prepared for first use in Illinois (state and federal § 1983 actions). Defaults are documented in [`docs/requirements.md`](docs/requirements.md) and are jurisdiction-tunable.
>
> **Not legal advice.** Statutes and rules of evidence are referenced for design context only. Admissibility, consent, retention, and disclosure decisions for any specific matter belong with licensed counsel.

## Release focus

Transcription, OCR, and scene detection are local, on-device engines (faster-whisper, pytesseract, scenedetect) that the desktop build does not bundle, so on the desktop they produced nothing. v0.1.16 adds the path users asked for: opt in under **Settings → AI**, supply an OpenAI-compatible base URL, API key, and model, and transcription runs in the cloud. Because the transcription API accepts audio *and* video directly, this needs no local ffmpeg and works on the desktop today. It is off by default; bundling the on-device engines for a fully-local desktop is the follow-up.

## Highlights

- **Bring-your-own-key cloud transcription (`backend/src/loom/services/transcription.py`, `workflows/transcription_activities.py`).** When the cloud engine is configured, the transcription pipeline sends the original asset to the provider and stores the returned segments. No ffmpeg, no local model. Diarization (speaker labels) remains a local-only feature.
- **An AI settings page (`frontend/src/routes/settings/ai.tsx`, `GET/PUT /api/v1/settings/ai`).** Choose the transcription engine (on-device vs cloud), set the base URL, model, and a masked API key, reachable from a new sidebar link. The page warns that cloud processing sends evidence off the machine. OCR and scene detection are shown as on-device only.
- **Egress is explicit and audited.** Cloud transcription is off by default and only an **admin** can change it. The API key is stored in the local database and never returned by the API (only whether one is set). The configured endpoint is validated to reject loopback/private targets. Every cloud transcription writes a `cloud_transcription` chain-of-custody entry (provider, model, endpoint) on the asset, and that provenance surfaces in the existing "Why?" popover.
- **New `app_settings` table (migration 013).** A small key-value store for runtime, admin-editable config; the lite profile materialises it via `create_all`, the server profile via the migration.

## Migrations

- **Alembic 013** adds the `app_settings` table. Server installs run `alembic upgrade`; Desktop Lite materialises it automatically on next launch. No factory reset, no data migration.

## Install

- **Desktop Lite (individuals):** download the installer for your platform from this release's assets, or follow [`docs/desktop-lite.md`](docs/desktop-lite.md) to build from source. No factory reset required.
- **Server (organizations):** see [`docs/prod-deploy.md`](docs/prod-deploy.md); run `alembic upgrade head` for migration 013.

## Known limitations in 0.1.16

- **On-device AI on the desktop is still pending.** Local transcription/OCR/scene-detection and the bundled `ffmpeg`/`tesseract` binaries are a follow-up; until then, transcription on the desktop requires the opt-in cloud path, and OCR/scene detection have no cloud option yet.
- **Cloud transcription covers transcription only.** OCR and scene detection have no cloud provider; speaker diarization is local-only (off by default per BIPA alignment in `docs/requirements.md`).
- **Sending evidence to a third-party API is a legal decision.** Loom records the egress in chain of custody but does not assess a provider's retention, training, or jurisdiction — that judgment belongs with counsel.
- **Three breaking-major frontend upgrades held for dedicated migration:** Vite 8, Tailwind CSS 4, ESLint 10. Carryover from v0.1.13.
- **macOS Intel installer is still not produced** (Apple Silicon only). Carryover from v0.1.1.
- **Desktop sidecar cold-start on Windows can still take 15–25 s.** Carryover from v0.1.3.

## Security

- Cloud transcription is **opt-in, off by default, admin-only**. The API key is persisted in the local database and never echoed back by the API. The configured endpoint is structurally validated to reject loopback/private/link-local hosts. Desktop Lite still makes no outbound connection unless the user submits a URL or has explicitly enabled cloud transcription; see the updated network-posture note in [`docs/security.md`](docs/security.md) and the [Lite capability matrix](docs/lite-capability-matrix.md). The documented suppressions remain CVE-2025-3000 (torch, local-only) and the temporalio-vendored pyo3 advisory in `.trivyignore`.
- Report vulnerabilities privately by emailing the maintainer — do not open a public issue.
