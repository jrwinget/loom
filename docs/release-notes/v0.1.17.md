Sixteenth beta release of Loom — an evidence operating system for civil-rights legal teams that turns mixed field evidence (video, photos, audio, FOIA returns) into a defensible event timeline where every claim traces back to source material. Civil-liberties tooling: no face recognition, no suspicion scoring, no automated identity resolution.

> **Beta scope:** prepared for first use in Illinois (state and federal § 1983 actions). Defaults are documented in [`docs/requirements.md`](docs/requirements.md) and are jurisdiction-tunable.
>
> **Not legal advice.** Statutes and rules of evidence are referenced for design context only. Admissibility, consent, retention, and disclosure decisions for any specific matter belong with licensed counsel.

## Release focus

Two fixes that make the desktop app usable for actual review. First, evidence media now displays: PDFs and video failed to render in the packaged desktop app because its Linux webview (WebKitGTK) has no built-in PDF viewer and ships without common video codecs — the signed-URL serving path was already correct; the rendering engine was the gap. Second, choosing a cloud transcription provider is now a guided provider → model pick instead of two freeform text boxes, with curated models per provider and a self-hosted option.

## Highlights

- **PDFs render in the desktop app via bundled pdf.js (`frontend/src/lib/pdf.ts`, `components/asset/asset-viewer.tsx`).** Pages render to a canvas with paging and zoom, replacing the `<object>` element that only worked where the webview had a native PDF viewer. CMap and standard-14 font data ship with the bundle so real-world PDFs render with correct glyphs; a download fallback shows on load/render failure. pdf.js loads lazily, on first use.
- **Video plays across every installer, with a graceful fallback.** The `.deb` declares the GStreamer codec plugins; the AppImage bundles them (`APPIMAGE_BUNDLE_GSTREAMER`); macOS (WKWebView) and Windows (WebView2) decode H.264/AAC natively. When a codec is unsupported, the viewer offers a download instead of a blank frame. The fix is shared frontend/Tauri config, so it applies uniformly to the dmg, msi, nsis, appimage, and deb bundles.
- **Guided AI provider/model picker (`frontend/src/routes/settings/ai.tsx`, `GET /api/v1/settings/ai/providers`).** A grouped provider dropdown drives a curated model dropdown, backed by a single source-of-truth catalog (`backend/src/loom/services/ai_providers.py`): OpenAI and Google Gemini (frontier), an open-source/self-hosted group, and a custom OpenAI-compatible option. Anthropic is listed but disabled — the Anthropic API does not accept audio input, so it cannot transcribe. Hosted base URLs are derived from the catalog; the self-hosted/custom options may target a local server.
- **Gemini transport (`backend/src/loom/services/transcription.py`).** Cloud transcription now dispatches by provider: the existing OpenAI-compatible `/audio/transcriptions` path, plus a Gemini `generateContent` adapter. Provenance and the `cloud_transcription` chain-of-custody entry record the chosen provider.

## Migrations

- **None.** The `app_settings` `ai` config gains an optional `provider` field, read with a default; a config saved before this release is treated as a custom OpenAI-compatible endpoint. No schema change, no factory reset.

## Install

- **Desktop Lite (individuals):** download the installer for your platform from this release's assets, or follow [`docs/desktop-lite.md`](docs/desktop-lite.md) to build from source. No factory reset required.
- **Server (organizations):** see [`docs/prod-deploy.md`](docs/prod-deploy.md); no new migration in this release.

## Known limitations in 0.1.17

- **Anthropic can't transcribe via API.** It is listed but disabled; for an open-weights Anthropic audio model, self-host it under the open-source option.
- **The Gemini path returns prose, not timed segments** — it is stored as a single full-file transcript segment (no per-line timestamps) — and inlines the file in the request (~15 MB cap); larger files need an OpenAI-compatible provider.
- **AppImage video codecs are bundled at build time.** Verify playback on a clean host with no system GStreamer; a web-friendly transcoded proxy (ffmpeg) is the longer-term fix and is not yet bundled.
- **On-device AI on the desktop is still pending.** Local transcription/OCR/scene-detection and the bundled `ffmpeg`/`tesseract` binaries are a follow-up; transcription on the desktop still requires the opt-in cloud path, and OCR/scene detection have no cloud option. Carryover from v0.1.16.
- **Three breaking-major frontend upgrades held for dedicated migration:** Vite 8, Tailwind CSS 4, ESLint 10. Carryover from v0.1.13.
- **macOS Intel installer is still not produced** (Apple Silicon only). Carryover from v0.1.1.
- **Desktop sidecar cold-start on Windows can still take 15–25 s.** Carryover from v0.1.3.

## Security

- Cloud transcription remains **opt-in, off by default, admin-only**. The API key is persisted in the local database and never echoed back by the API. Hosted-provider endpoints are derived from the catalog; the loopback/private-host block is lifted only for the explicitly self-hosted and custom providers (whose purpose is a local/LAN server), while the `http(s)` structural check always applies. Desktop Lite makes no outbound connection unless the user submits a URL or has enabled cloud transcription — now documented as a second opt-in egress in [`docs/desktop-lite.md`](docs/desktop-lite.md) and the [Lite capability matrix](docs/lite-capability-matrix.md). Documented suppressions are unchanged (torch CVE, local-only; the temporalio-vendored pyo3 advisory in `.trivyignore`).
- Report vulnerabilities privately by emailing the maintainer — do not open a public issue.
